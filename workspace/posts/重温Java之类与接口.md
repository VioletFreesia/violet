# 重温Java之类与接口
> 注意: 知识点整理基于Java11, 不保证以前的版本可用
## 一些常识

- 成员变量由`field`一词意译过来
- Java里的属性(property)一般指的是`setter`和`getter`方法
- `this`总是指向调用该方法的对象
- 封装: 不允许外部直接访问成员变量
- 继承: `extends`,原意为扩展,子类是对父类的扩展, 子类不能获得父类的构造器
- 多态: 一个变量有两个类型,一个是编译时类型,一个是运行时类型, 两个类型不一致时就会出现多态
  如子类对象赋值给父类对象,运行时又强转为子类对象,则编译时是父类类型,运行时是子类类型
- `super`限定: `super`表示调用父类的方法,`super()`为父类构造器
- `a instanceof A`: 判断a是不是A或A的子类的实例

## 静态代码块,构造代码块,构造函数

构造函数:

```java
public HelloWorld(){
    //code here
}
```

- **对象一建立，就会调用与之相应的构造函数**，不建立对象，构造函数是不会运行的。
- 构造函数的作用是用于给对象进行初始化。
- .一个对象建立，构造函数只运行一次，而一般方法可以被该对象调用多次。

构造代码块:

```java
{
    //code here
}
```

- 构造代码块的作用是给对象进行初始化。
- **对象一建立就运行构造代码块了，而且优先于构造函数执行**。有对象建立，才会运行构造代码块，类不能调用构造代码块
- 构造代码块与构造函数的区别是：**构造代码块是给所有对象进行统一初始化，而构造函数是给对应的对象初始化**，因为构造函数是可以多个的，运行哪个构造函数就会建立什么样的对象，但无论建立哪个对象，都会先执行相同的构造代码块。也就是说，构造代码块中定义的是不同对象共性的初始化内容。

静态代码块:

```java
static {
    //code here
}
```

- 它是**随着类的加载而执行，只执行一次，并优先于主函数**。具体说，**静态代码块是由类调用**的。类调用时，先执行静态代码块，然后才执行主函数的。
- **静态代码块其实就是给类初始化的，而构造代码块是给对象初始化的**。
- 静态代码块中的变量是局部变量，与普通函数中的局部变量性质没有区别。
- 一个类中可以有多个静态代码块

## final关键字

- 修饰的变量只能赋值一次
- 修饰的方法不可被重写
- 修饰的类不能被继承

## 抽象类与接口

- 二者皆不能实例化
- 抽象类与普通类的区别就是抽象类可能存在抽象方法
- 抽象类与抽象方法都要用abstract修饰
- 接口可以拥有的成员为: 静态常量, 抽象方法, 类方法, 默认方法, 私有方法(Java9+), 内部类(包括内部接口, 枚举)
- 接口里的类方法, 默认方法, 私有方法必须有方法体
- 接口的默认方法其实就是实例方法, 必须使用default修饰,且不能使用static修饰
- 接口常用于定义标准; 抽象类常用作实现某个功能的中间件,体现的是模板式设计

## 内部类

- 内部类的定义: 把一个类的定义放在另一个类的内部即可, 其中"类内部"可以是类中的任何位置
  包括方法中(局部内部类)
- 静态内部类: 使用static修饰,该类就属于外部类本身,而不是属于外部类的某个实例对象
- 非静态内部类:  属于外部类的某个实例对象,需要通过实例对象才能调用
- 局部内部类: 在方法中定义类, 该类只在该方法内有效
- 匿名内部类: 创建只需要使用一次的类, 一般会在实现该类的同时创建实现类的一个实例, 由于是直接创建的实例对象, 没有类名, 所以匿名内部类无法复用
  - 匿名内部类其实是实现的某个接口或继承的某个父类
  - 匿名内部类不能是抽象类

## Lambda表达式

Lambda表达式其实就是一种语法糖, 他是匿名内部类实现的一种简写

当匿名内部类的父类或者实现的接口只有一个抽象方法时(统称为函数式接口),就可以使用lambda表达式来进行匿名内部类对象的创建

- 方法引用与构造器引用(用其他方法来代替需要重写的抽象方法)

|          种类          |        示例        |                             说明                             |           对应的lambda表达式            |
| :--------------------: | :----------------: | :----------------------------------------------------------: | :-------------------------------------: |
|       引用类方法       |    类名::类方法    |    函数式接口中被实现的方法的全部参数传给该类方法作为参数    |    (a,b,...) -> 类名.类方法(a,b,...)    |
| 引用特定对象的实例方法 | 特定对象::实例方法 |     函数式接口中被实现的方法的全部参数传给该方法作为参数     | (a,b,...) -> 特定对象.实例方法(a,b,...) |
| 引用某类对象的实例方法 |   类名::实例方法   | 函数式接口中被实现的方法的第一个参数作为调用者,后面的参数全部传给该方法作为参数 |    (a,b,...) -> a.实例方法(a,b,...)     |
|       引用构造器       |     类名:: new     |    函数式接口中被实现的方法的全部参数传给该构造器作为参数    |     (a,b,...) -> new 类名(a,b,...)      |


- 每个枚举对象可以有自己的方法
